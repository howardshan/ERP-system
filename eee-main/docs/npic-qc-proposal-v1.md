## 一、执行摘要

NPIC 目前的生产与质检流程依赖纸质记录，导致质检结果滞后、夜班决策缺失、部门间信息不对称三大核心问题。这些问题直接造成包装产能浪费、库存数据失真，以及仓库与包装部门之间的协调摩擦。

本提案将以 **QC 实时管理系统（Phase 1）** 为切入点，将 NPIC 现有的纸质烘干记录、取样检验流程和管理层决策全面数字化，实现当班即可查看检验结果、合格品自动进入包装队列、管理层远程实时掌控生产状态。

**Phase 2** 将在 Phase 1 数据基础上，打通包装出库 → 成品仓 → 工单销单的完整链路，彻底解决仓库与包装部门的数据对账问题。

## 二、现状问题诊断

### 2.1 质检信息严重滞后

NPIC 目前的检验流程是：

> 生产线为每辆小车贴标 → 推入烘干房 → 班中取样 → **当班结束前完成全部检测** → 结果汇报给管理楼层 → 管理层决定是否安排包装

**核心问题：检测必须等到当班即将结束时才能全部完成，结果无法在班中实时获取。**

这意味着：

- 每辆车从烘干完成到检测结果出来，中间存在数小时的信息空窗期；班中无法根据检测结果做任何实时决策；
- 夜班没有管理人员在岗，夜班的检验结果无人即时处置，包装和调配工作被迫压缩至次日早班，造成早班极度紧张；
- 产线上出现的品质问题（如水活性超标批次）无法实时干预，问题批次可能继续流转，增大损失风险。

### 2.2 夜班管理真空

由于检测结果只在班末汇总，管理层无法在夜班时远程判断哪些批次合格、哪些批次需要处置。夜班工人缺乏明确指引，合格品堆积等待，导致次日早班的包装工作量剧增，造成：

- 包装部门突然接到大批量订单但仓储准备不足；
- 为完成出货，不得不直接从生产线提取尚未经过正规入库的货物；
- 仓库系统中显示为在制（CIP）状态的批次实际已被发运，库存账实不符。

### 2.3 跨部门信息断层

仓库说「已发出」，包装说「未收到」——这一矛盾反复发生，根本原因是两个部门之间没有共享的实时数字记录：

- 批次的流转状态只存在于各自的纸质记录或口头沟通中；
- 无法追溯某一批次从烘干完成到包装出库的完整链路；
- 一旦出现纠纷，无凭据可查，问题归因困难，部门矛盾积累。

## 三、Phase 1 解决方案：QC 实时管理系统

### 3.1 方案概述

我们将为 NPIC 构建一套专属的 QC 实时管理软件，覆盖从生产批次创建到检验结果决策的完整流程，取代现有的全部纸质记录。

系统核心流程如下：

```
生产批次录入（工单扫码）
        ↓
烘干房签入 / 分配烘干位
        ↓
到期提醒 → 取样 → 水活性检测 → 系统自动判定 Pass / Fail
        ↓
  ┌─── Pass ───────────────────────────────┐
  │                                        ↓
  │                              合格品进入「待包装队列」
  │                           管理层一键放行 → 进入 Phase 2 包装流程
  │
  └─── Fail ───────────────────────────────┐
                                           ↓
                                     处置选择：
                               复测 / 重烘 / 常温烘干 / 报废
                                           ↓
                              对应物料流转（返烘 / 等待 / 关闭）
```

### 3.2 核心功能模块

#### 模块 A — 生产批次与小车管理

- 工单扫码创建生产批次，系统自动生成小车编号（`工单号-001` 至 `工单号-N`）
- 每辆小车的标签信息与系统记录绑定，扫码即可查询实时状态

#### 模块 B — 烘干房实时监控

- 实时显示各烘干房的在烘车辆及状态
- 基于设定的目标烘干时长自动计算预计完成时间
- 到期自动提醒，避免过烘或超时未取

#### 模块 C — 取样与检验流程

- 支持抽样比例设置（如每 N 辆车取 1 个代表样），减少不必要的检验次数
- 取样 ID 与系统绑定，检验员录入水活性值后系统**自动判定 Pass / Fail**（依据各 SKU 的标准限值）
- 检验记录**实时可查**，无需等到班末汇总

#### 模块 D — 不合格品处置工作流

- 失败批次直接在系统中选择处置方式（复测 / 返烘 / 常温烘干 / 报废）
- 处置决策有据可查，全程留痕，质量追溯完整

#### 模块 E — 管理仪表盘

- 实时统计：当前在烘数量 / 待取样 / 待检验 / 今日合格率 / 需关注批次
- 合格品一键放行，进入包装队列

#### 模块 F — 批次全程追溯

- 任意批次可查：生产批 → 子批（小车）→ 取样记录 → 检验结果 → 处置决定 → 放行时间
- 部门间出现争议时，系统提供完整证据链

### 3.3 解决的核心问题对照

| 现有问题 | Phase 1 解决方式 |
|----------|-----------------|
| 检测结果只在班末汇总，班中无法决策 | 检验员现场录入，结果即时可见 |
| 包装部门无法预判入库量 | 合格品放行后立即进入待包装队列，可提前准备 |
| 问题批次发现滞后，无法当班处置 | 检验失败即时触发处置流程，当班内解决 |

## 四、Phase 2 愿景：全流程数字化（待议）

Phase 2 将在 Phase 1 质检数据的基础上，向前后两端延伸：

### 向下延伸：包装 → 成品仓

- 合格品放行后，系统自动生成包装任务
- 包装完成扫码确认，成品入库记录自动生成
- 成品仓库存实时更新，消除「仓库说发了、包装说没收到」的信息黑洞

### 向上延伸：工单 → 出库 → 订单销单

- 关联客户订单，出库时自动匹配工单
- 支持「走单」模式（为某客户专项生产专项发运）
- 库存状态实时可查：在制 / 待包装 / 成品库 / 已发运

**Phase 2 的目标是建立一条从「生产计划」到「客户收货」的完整数字链路，让 NPIC 每一批货物都可追、可查、可管。**

本系统为专属定制软件，运行于公司电脑及平板设备，所有数据实时同步、集中存储：

| 方面 | 说明 |
|------|------|
| 运行设备 | Windows / Mac 电脑及平板均可使用，界面自适应屏幕尺寸 |
| 数据存储 | 云端数据库，数据实时备份，断网后恢复连接自动同步 |

| 部署维护 | 由我方负责部署与日常维护，NPIC 无需配置服务器或 IT 人员 |

## 七、预期收益

### 运营效率提升

- 检验结果实时可见，**决策时效从「班末汇报」缩短至「检测即时可见」**

### 质量管控加强

- 不合格批次在当班内完成处置，避免问题批次流转到下一环节
- 完整的检验历史数据为长期品质分析和工艺改进提供基础

### 部门协同改善

- 计划、质检、包装三个部门共享同一套实时数据
- 批次全程追溯，部门纠纷有据可查
- 减少因信息不对称引发的沟通成本和部门摩擦
