一、概述

本提案阐述了 NPIC 数字化运营平台第二阶段的工作范围。在第一阶段（模块 A–I）成功建立 QC 检测系统的基础上，第二阶段专注于生产车间的数字化，以实时、权限管控的系统取代现有的 Excel 日报和纸质车辆记录，并与 Sage 100 直连以消除人工重复录入。

第二阶段新增五个模块（A–E）：

模块名称用途
A生产 — 主管批量录入在特殊情况下，主管批量录入员工产出，替代 Excel 日报
B生产 — 员工站点模式员工在机器旁实时逐车录入
C烘房操作车辆扫码出入库，与模块 B 的生产记录自动关联
D资料总计与报表生产数据汇总、效率报表及数据导出
ESage 100 集成NPIC 系统与 Sage 100 之间的实时双向数据同步

二、现状与问题

业务领域当前方式主要痛点
生产追踪Excel 日报（2026 Daily Report Forming Production.xlsx）手工录入——每位主管每天约需 2 小时录入；容易出错；无实时可见性；效率指标需手动计算；无法追踪车次生产时间
烘房纸质车辆记录无法系统追踪车辆链，半车接续跨班记录混乱
资料汇总Excel 手动汇总无统一报表视图；提取班次汇总或效率趋势需额外人工整理
Sage 100 数据录入所有生产记录由员工手动逐条录入 Sage 100重复录入耗时耗力，转录易出错；生产与系统数据之间存在延迟

三、系统架构

第二阶段基于与第一阶段相同的技术栈构建，并与现有 NPIC 平台直接集成。新模块共享第一阶段已建立的用户认证、条码扫描和数据导出基础设施。

角色与权限设计

角色可访问模块需要 MFA？
员工模块 B — 站点模式（车辆录入）否
主管模块 A、B、C、D — 完整录入、审核与报表否（标准操作）
管理员所有模块 + 受限操作（关单）是 — Microsoft Authenticator
系统管理员完整系统权限，包括 Sage 100 集成配置是 — Microsoft Authenticator

核心设计原则

  • 员工模式 — 专为生产车间使用设计的轻量界面，日常录入无需 MFA，支持刷卡或输入工号打卡。
  • 主管模式 — 完整的批量数据录入能力，界面布局与现有 Excel 表单一致。
  • 受限操作 — 关单需 Microsoft Authenticator 验证（继承自第一阶段模块 I）。
  • 按 SKU 配置字段 — 必填字段（重量 vs 数量、车辆层数、必填工单字段）按产品 SKU 预先配置，录入时自动生效。
  • 半车关联 — 在生产环节（模块 B）完成绑定后，烘房、报表、Sage 100 同步等所有下游模块自动继承该关联，无需重新确认。

四、模块规格说明

模块 A  生产 — 主管批量录入

替代现有 Excel 日报。主管可批量录入某一班次或时段内所有员工和车辆的记录。该模式适用于没有独立条码扫描站的团队。

采集数据
  • 员工姓名及工号
  • 机器分配情况
  • 工单号
  • 一级单（原材料编号）
  • 车号、层数及每车数量或重量
  • 班次时长与总工时
输出
  • 自动计算各员工效率指标（汇入模块 D 报表）
  • 记录与工单关联，供模块 E 同步至 Sage 100
模块 B  生产 — 员工站点模式

面向员工的实时生产录入界面，在机器或产线旁的共享平板 / 终端上使用。日常录入无需 Microsoft Authenticator。

签到 / 签出
  • 员工在开工前通过输入工号或刷卡签到；同一机器可多名员工同时签到。
  • 签出时需进行电子签名。
  • 产线所有员工签名后，本班次自动结束。
车辆录入
  • 录入第一车时，需输入一级工单号；此后各车自动默认上次输入的工单号，但必须按确认按钮方可继续，手动输入可随时覆盖默认值。
  • 一辆车可关联多个一级工单号。
  • 必填字段（重量或数量、层数）按产品 SKU 预配置，自动生效。
  • 每条记录需录入车号及层数。
半车接续
  • 录入本班第一车时，员工必须选择以下之一（必填项，不可跳过）：
  • 独立新车 — 本班次从头开始的新一车生产。
  • 半车接续 — 本车为上一次班次某车的物理接续，系统将当前车号与上一车号关联。烘房、报表及 Sage 100 同步等所有下游模块自动继承该关联，无需在任何其他环节重新确认。
关单 — 受限操作（需 MFA）

由于实际生产数量通常小于理论 BOM 数量，系统提供关单功能。该操作需要特殊权限和 Microsoft Authenticator 验证。

选项说明
无剩余物料已全部用尽，工单干净关闭，无剩余原材料。
剩余物料作废剩余原材料报废处理，系统自动将未用物料数量平均分配至已完成的各车记录，用于精确物料核算。
保留剩余物料剩余物料留存，供下次生产使用，各车记录无需调整。
模块 C  烘房操作

记录车辆进出烘房，并与生产记录自动关联。半车绑定关系已在模块 B 中建立，烘房无需重新确认。

  • 扫码入库：操作员扫描车辆条码，系统自动从生产记录中提取该车信息（含半车绑定关系）并记录入库时间戳。
  • 扫码出库：记录车辆离开烘房的时间戳。

其余烘房工作流细节（返烘队列集成、时间 / 温度记录等）将在需求确认阶段与 NPIC 团队共同敲定。

模块 D  资料总计与报表

将模块 A 和模块 B 录入的所有生产数据汇聚成结构化报表和导出文件，替代现有的 Excel 手动汇总流程。

班次与日度报表
  • 自动生成班次汇总：每班总车数、总产量 / 重量、总工时、在线人数
  • 按日汇总：全机台、全工单的当日生产情况
  • 工单完工报表：计划量 vs 实际量、物料核算状态
效率报表
  • 各员工产出指标（完成车辆数、重量 / 产量、工时），支持任意日期范围筛选
  • 机台利用率：各产线有效工时 vs 停机时长
  • 趋势分析：按产品或工单查看周度 / 月度产出对比
数据导出
  • 所有报表均可按日期范围或产品 SKU 导出
  • 导出格式：Excel 及 PDF
  • 复用第一阶段导出框架，保持一致性
模块 E  Sage 100 集成

NPIC 系统直连 Sage 100 数据库，实现实时双向数据同步，彻底消除工单、库存调整和生产完工数据的手动重复录入。

同步设计
  • 实时拉取：用户开立新记录时按需从 Sage 100 拉取工单及物料主档数据，确保始终使用最新信息。
  • 事件触发推送：生产完工和库存调整在关单等关键工作流节点触发，而非定时轮询，最大程度减少数据冲突。
  • 错误处理：推送失败时系统自动排队重试；对于需要人工处理的条目，主管会收到通知提醒。

详细说明

7.1 消除手工 Excel 日报　现状：各主管每天手工在 Excel 汇总当班生产、编制日报；系统上线后由现场实时采集自动生成，无需再手动录入。按每位主管每天填报约 2 小时、10 位主管 × 22 个工作日计，每月约节省 440 工时（≈2.5 个全职人力）；日报由「次日可见」变为「实时可见」。

7.2 实时车辆级数据　每车完成即生成带时间戳的记录，无需事后凭记忆补录。事后补录量趋近 0；每车可追溯率 100%，时间戳精确到分钟级。

7.3 工单级原材料用量精准追踪　按每个工单记录实际领用与消耗。现状痛点：一个工单的料常被用到另一个工单上，导致实际用量无法准确归集；系统按工单绑定用料后，每单实际用量可准确核算。消除跨工单串料导致的用量失真，每单实际用量 / 单位产出用料（yield）可准确核算，支撑成本核算与超耗识别。

7.4 车次产量与时间精准追踪　每车实际生产数量 + 开始 / 完成时间点由系统实时采集。现状痛点：存在员工故意漏车 / 瞒报，实际生产车数与登记车数不一致；系统实时计数后两者一致。实际车数 = 登记车数，漏报车次归零、产量登记准确率提升至 100%；可算平均节拍（分钟 / 车）用于产能测算与排产。

7.5 车次生产后等待时长记录　记录每一车生产完成后的等待时长（如等待进入下一环节 / 烘干房），持续积累数据，为后续流程优化、效率提升提供依据。平均等待时长（分钟 / 车）可统计，作为效率改善的量化基线与追踪指标。

7.6 免手工录入工位与工时　员工工位分配与工作时长由系统采集，替代纸面 / Excel 手工填报。此项的工时节省已包含在 7.1 的日报节省中，不单独重复计量；价值在于工时 / 工位数据的完整率与准确率提升（消除漏填、错填）。

7.7 员工车次生产效率　按「员工 × 车次」由系统自动、准确核算生产效率（车 / 班、件 / 工时），比现有方式更精细、更客观、可实时。效率数据颗粒度到「人 × 车」；可比较班组 / 个人效率离散度，支撑绩效评估与人员调配。（现已有客观基线，此项为「更准更细的效率数据」，非从无到有。）

7.8 工单级实时生产追踪　实时追踪每个工单的生产进度 / 实况，计划与生产总监可据实效进行更准确的监控与规划（排产、产能、交期）。工单进度实时可见（覆盖率 100%），从「事后汇总」变为「实时掌握」，支撑更准确的排产与产能决策。

7.9 免手工填写每车记录与统计报表（产线员工）　现状：产线员工需逐车手动填写每车生产记录及统计报表；系统上线后由现场实时采集，员工无需再手动填写。按 1 分钟 / 车 × 6 车 / 班 × 3 班 / 天 × 40 条产线 = 720 分钟 / 天（12 工时 / 天），22 个工作日计，每月约节省 264 工时（≈1.5 个全职人力）。

1. Overview

This proposal outlines the scope of Phase 2 of the NPIC digital operations platform. Building on the QC inspection system established in Phase 1 (Modules A–I), Phase 2 focuses on digitizing the production floor — replacing the daily Excel log and paper-based cart records with a real-time, role-controlled system, while integrating directly with Sage 100 to eliminate manual double-entry.

Phase 2 introduces five modules (A–E):

ModuleNamePurpose
AProduction — Supervisor Data EntryBatch logging of employee output by supervisors, replacing the Excel daily report
BProduction — Employee Station ModeReal-time cart-by-cart entry by employees at the machine, with sign-in/out
CDry Room OperationsCart scan-in/out linked to production records from Module B
DData Summary & ReportingAggregated shift reports, productivity metrics, and data exports
ESage 100 IntegrationReal-time bi-directional data sync between the NPIC system and Sage 100

2. Current State & Challenges

3. System Architecture

Phase 2 is built on the same technical stack as Phase 1 and integrates directly into the existing NPIC platform. New modules share the same user authentication, barcode scanning, and export infrastructure.

Role & Access Design

Key Design Principles

  • Employee Mode — Lightweight production-floor interface. Badge scan or employee ID sign-in. No MFA required for daily entries.
  • Supervisor Mode — Full batch data entry matching the layout of the existing Excel form.
  • Privileged Actions — Work order closing requires Microsoft Authenticator verification (inherited from Phase 1, Module I).
  • SKU-Configurable Fields — Required fields (weight vs. quantity, cart tray layers, work order fields) are pre-configured per product SKU.
  • Half-Cart Linking — Physical cart continuations are bound in the production stage (Module B); all downstream modules inherit the association automatically.

4. Module Specifications

Module A  Production — Supervisor Data Entry

Replaces the existing Excel daily report. Supervisors enter batch records for all workers and carts in a given shift or period. This mode is used by teams that do not have individual barcode scanning stations.

Data Captured
  • Employee names and IDs
  • Machine assignments
  • Work order numbers (工单号)
  • Primary material codes — 一级单 (raw material identifiers)
  • Cart numbers, tray layers, and quantity or weight per cart
  • Shift hours and total labor time
Outputs
  • Auto-calculated per-employee productivity metrics (fed into Module D reporting)
  • Records linked to work orders for Sage 100 sync via Module E
Module B  Production — Employee Station Mode

A streamlined, employee-facing interface for real-time production logging at the machine or line level. Designed for shared tablets or terminals on the production floor. No MFA required for standard entries.

Sign-In / Sign-Out
  • Employees sign in before the shift by entering their employee ID or scanning their badge. Multiple employees can sign in simultaneously on the same machine.
  • A digital signature is required at sign-out.
  • The shift closes when all employees assigned to that line have signed out.
Cart Entry
  • On the first cart, the employee inputs the primary work order number(s). Subsequent carts auto-populate from the last entry — a confirmation button must be pressed before proceeding; manual input can override the default at any time.
  • One cart can be associated with multiple primary work orders.
  • Required fields (weight or quantity, tray layers) are pre-configured per product SKU.
  • Cart number and tray layer count are recorded for each entry.
Half-Cart Continuation
  • For the first cart of a session, the employee must select one of the following — mandatory, cannot be skipped:
  • Standalone cart — a new, independent production run.
  • Half-cart (continuing) — a physical continuation of a previous session's cart. The system links this cart to the prior cart number. All downstream modules (dry room, reporting, Sage 100 sync) inherit this association automatically — no re-selection required anywhere else in the workflow.
Work Order Close — Restricted (MFA Required)

Because actual production output is typically less than the theoretical BOM quantity, a work order close function is provided. This action requires special permission and Microsoft Authenticator verification.

OptionDescription
No RemainderAll materials fully consumed. Work order closes cleanly.
Scrap RemainderRemaining raw material written off. System auto-distributes unused quantity evenly across finished carts for accurate material accounting.
Retain RemainderRemaining material reserved for a future run. No adjustment to cart records.
Module C  Dry Room Operations

Tracks carts through the dry room with production-record linkage. No half-cart re-selection is required — the association established in Module B carries through automatically.

  • Scan-In: Operator scans the cart barcode on entry. The system retrieves the cart's production record (including any half-cart binding) automatically from the barcode and records the entry timestamp.
  • Scan-Out: Records exit timestamp when the cart leaves the dry room.

Additional dry room workflow details (redry queue, time/temperature logging) will be confirmed with the NPIC team during the requirements phase.

Module D  Data Summary & Reporting

Detailed Notes

7.1 Eliminate manual Excel daily report. Today, each supervisor manually consolidates the shift's production in Excel and compiles a daily report. Once the system is live, the report is generated automatically from real-time shop-floor capture, with no manual entry. At roughly 2 hours/day per supervisor, across 10 supervisors × 22 working days, this saves about 440 labor-hours/month (≈2.5 FTE); the daily report shifts from “available next day” to “available in real time.”

7.2 Real-time cart-level data. Each cart generates a timestamped record the moment it is completed — no after-the-fact reconstruction from memory. Backfilling approaches 0; per-cart traceability reaches 100%, with timestamps accurate to the minute.

7.3 Work-order-level raw-material usage tracking. Actual issuance and consumption are recorded per work order. Current pain point: material from one work order is often used on another, so actual usage cannot be attributed accurately. Binding material to the work order eliminates the distortion caused by cross-work-order material mixing, allowing each order's actual usage / yield to be computed accurately — supporting cost accounting and over-consumption detection.

7.4 Accurate cart count & timing tracking. Each cart's actual output quantity plus its start/finish timestamps are captured in real time. Current pain point: employees may deliberately omit or under-report carts, so the actual number produced does not match the registered number. With real-time counting the two match. Actual cart count = registered cart count, under-reporting drops to zero, output-registration accuracy rises to 100%, and an average takt time (minutes/cart) can be derived for capacity estimation and scheduling.

7.5 Post-production waiting-time logging. The waiting time after each cart finishes production (e.g., waiting to enter the next step / the drying room) is recorded, building up data that provides a basis for future process optimization and efficiency improvement. The average waiting time (minutes/cart) can be measured and used as a quantified baseline and tracking metric for improvement.

7.6 No manual entry of workstation & labor hours. Workstation assignment and working hours are captured by the system, replacing paper/Excel manual entry. The labor-hour saving here is already included in the 7.1 daily-report saving and is not double-counted; the value lies in improved completeness and accuracy of the labor-hour / workstation data (eliminating missing and incorrect entries).

7.7 Employee cart-level productivity. Productivity is computed automatically and accurately per employee × cart (carts/shift, units/labor-hour) — finer-grained, more objective, and real-time than the current approach. Data granularity reaches “person × cart”; productivity dispersion across teams/individuals can be compared, supporting performance evaluation and staffing decisions. (An objective baseline already exists; this item makes the data more accurate and more granular, not something created from nothing.)

7.8 Work-order-level real-time production tracking. The production progress/status of each work order is tracked in real time, so planning and production directors can monitor and plan more accurately based on actual performance (scheduling, capacity, delivery dates). Work-order progress is visible in real time (100% coverage), shifting from “after-the-fact roll-up” to “real-time command.”

7.9 No manual filling of per-cart records & reports (line workers). Today, line workers must manually fill in each cart's production record and statistical reports; once the system is live, data is captured in real time on the shop floor and workers no longer fill anything in manually. At 1 minute/cart × 6 carts/shift × 3 shifts/day × 40 production lines = 720 minutes/day (12 labor-hours/day), across 22 working days, this saves about 264 labor-hours/month (≈1.5 FTE).

Módulo B  Producción — Modo estación del empleado

Interfaz optimizada, orientada al empleado, para el registro de producción en tiempo real a nivel de máquina o línea. Diseñada para tabletas o terminales compartidas en el piso de producción. No requiere MFA para la captura estándar.

Entrada / salida
  • Los empleados inician sesión antes del turno ingresando su ID de empleado o escaneando su gafete. Varios empleados pueden iniciar sesión simultáneamente en la misma máquina.
  • Se requiere una firma digital al cierre de sesión.
  • El turno se cierra cuando todos los empleados asignados a esa línea han cerrado sesión.
Captura de carros
  • En el primer carro, el empleado ingresa el/los número(s) de orden de trabajo primaria. Los carros siguientes se llenan automáticamente a partir del último registro —debe presionarse un botón de confirmación antes de continuar; la captura manual puede sobrescribir el valor por defecto en cualquier momento.
  • Un carro puede asociarse a varias órdenes de trabajo primarias.
  • Los campos requeridos (peso o cantidad, capas de charola) se preconfiguran por SKU de producto.
  • Se registran el número de carro y el número de capas de charola en cada captura.
Continuación de medio carro
  • Para el primer carro de una sesión, el empleado debe seleccionar una de las siguientes opciones —obligatorio, no se puede omitir:
  • Carro independiente — una corrida de producción nueva e independiente.
  • Medio carro (continuación) — una continuación física del carro de una sesión anterior. El sistema vincula este carro con el número del carro previo. Explicaciones detalladas

7.1 Eliminar el reporte diario manual en Excel. Hoy, cada supervisor consolida manualmente en Excel la producción del turno y elabora un reporte diario. Con el sistema en operación, el reporte se genera automáticamente a partir de la captura en tiempo real en piso, sin captura manual. A razón de aproximadamente 2 horas/día por supervisor, con 10 supervisores × 22 días hábiles, se ahorran cerca de 440 horas-hombre/mes (≈2.5 FTE); el reporte diario pasa de «disponible al día siguiente» a «disponible en tiempo real».

7.2 Datos en tiempo real a nivel de carro. Cada carro genera un registro con marca de tiempo en cuanto se completa, sin reconstrucción posterior de memoria. La captura posterior se aproxima a 0; la trazabilidad por carro alcanza el 100%, con marcas de tiempo con precisión al minuto.

7.3 Seguimiento del uso de materia prima por orden de trabajo. La entrega y el consumo reales se registran por cada orden de trabajo. Problema actual: con frecuencia el material de una orden se usa en otra, por lo que el consumo real no puede atribuirse con exactitud. Al vincular el material a la orden de trabajo se elimina la distorsión causada por la mezcla de material entre órdenes, permitiendo calcular con exactitud el consumo real / rendimiento (yield) de cada orden, lo que respalda el costeo y la detección de sobreconsumo.

7.4 Seguimiento exacto de cantidad y tiempos por carro. La cantidad real producida de cada carro y sus marcas de tiempo de inicio/fin se capturan en tiempo real. Problema actual: los empleados pueden omitir o subreportar carros deliberadamente, por lo que la cantidad realmente producida no coincide con la registrada. Con el conteo en tiempo real ambas coinciden. Carros reales = carros registrados, el subreporte baja a cero, la exactitud del registro de producción sube al 100% y puede derivarse un tiempo takt promedio (minutos/carro) para estimar capacidad y programar.

7.5 Registro del tiempo de espera tras la producción. Se registra el tiempo de espera después de que cada carro termina su producción (por ejemplo, la espera para pasar a la siguiente etapa / al cuarto de secado), acumulando datos que sirven de base para la optimización del proceso y la mejora de la eficiencia a futuro. El tiempo de espera promedio (minutos/carro) puede medirse y usarse como línea base cuantificada y métrica de seguimiento para la mejora.

7.6 Sin captura manual de estación y horas de trabajo. La asignación de estación y las horas de trabajo las captura el sistema, reemplazando la captura manual en papel/Excel. El ahorro de horas de este punto ya está incluido en el ahorro del reporte diario de 7.1 y no se contabiliza dos veces; el valor está en la mayor integridad y exactitud de los datos de horas/estación (eliminando faltantes y errores de captura).

7.7 Productividad del empleado por carro. La productividad se calcula de forma automática y exacta por empleado × carro (carros/turno, piezas/hora-hombre); más detallada, objetiva y en tiempo real que el método actual. La granularidad de los datos llega a «persona × carro»; puede compararse la dispersión de productividad entre equipos/individuos, apoyando la evaluación de desempeño y las decisiones de asignación de personal. (Ya existe una línea base objetiva; este punto hace los datos más exactos y más detallados, no algo creado desde cero.)

7.8 Seguimiento de producción en tiempo real por orden de trabajo. Se sigue en tiempo real el avance/estado de producción de cada orden de trabajo, de modo que los directores de planeación y de producción puedan monitorear y planear con mayor precisión con base en el desempeño real (programación, capacidad, fechas de entrega). El avance de la orden es visible en tiempo real (100% de cobertura), pasando de una «consolidación posterior» a un «control en tiempo real».

7.9 Sin llenado manual de registros y reportes por carro (operadores de línea). Hoy, los operadores de línea deben llenar manualmente el registro de producción y los reportes estadísticos de cada carro; con el sistema en operación, los datos se capturan en tiempo real en piso y los operadores ya no llenan nada manualmente. A razón de 1 minuto/carro × 6 carros/turno × 3 turnos/día × 40 líneas de producción = 720 minutos/día (12 horas-hombre/día), con 22 días hábiles, se ahorran cerca de 264 horas-hombre/mes (≈1.5 FTE).
