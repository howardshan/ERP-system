1. Executive Summary

NPIC's current production and quality inspection processes rely on paper records, resulting in three core problems: delayed QC results, absence of decision-making support during night shifts, and information asymmetry between departments.

This project will build a dedicated QC Real-Time Management System (Phase 1) for NPIC, covering the complete workflow from production batch creation to inspection result decisions and qualified product release to the packaging queue — replacing all existing paper records.

System development is complete. Both parties have concluded the final requirements meeting and are now entering the testing and go-live phase. This document details the system scope, testing plan, implementation plan, and investment.

2. Current State Diagnosis

2.1 QC Information Is Significantly Delayed

All tests can only be completed near the end of each shift; results are not available in real time. This means:

  • An information gap of several hours between when a cart finishes drying and when results are available;
  • Quality issues cannot be addressed in real time; problem batches may continue to flow downstream;
  • Drying room check-in records depend on manual paper forms, causing delays and errors.

2.2 Night Shift Decisions Lack Real-Time Support

Because test results are only summarized at end of shift, management cannot remotely determine during the night shift which batches are qualified and which require disposition.

2.3 Cross-Department Information Silos

The warehouse and packaging teams lack shared real-time records. Batch flow status exists only in each department's paper records or verbal communication. When disputes arise, there is no evidence to consult.

2.4 Drying Room Check-In/Out Relies on Manual Paper

Cart number entries are prone to duplication errors. Every morning shift, staff must manually transcribe check-in forms for all pending drying carts — taking over 3 hours — with frequent data errors.

2.5 Sales Team Cannot Access Real-Time Production Status

Because the various systems are not connected, the sales team cannot monitor production progress and inventory status in real time, making delivery commitments and inventory planning unreliable.

3. System Scope: QC Real-Time Management System

3.1 System Workflow

The system covers the complete workflow:

  • Work order creation → Label printing → Production staff attach labels to carts
  • Barcode scanner scan → Confirm cart entry into drying queue
  • Manual drying room assignment → Automatic countdown timer starts
  • Expiry alert → Batch checkout → Sampling notification
  • Lab technician enters water activity reading → System auto-judges Pass / Fail
  • Qualified products enter packaging queue → Management releases with one click
  • Non-conforming products trigger disposition workflow (Re-test / Re-dry / Scrap)
  • Packaging dept views all released carts, multi-selects, batch dispatches

3.2 Core Functional Modules

Module A — Production Batch & Cart Management

  • Work order creation generates production batches; system auto-generates cart numbers (WO#-001 through WO#-N)
  • Each cart label is bound to its system record; scan to view real-time status
  • Work orders can be printed; production staff attach labels to corresponding carts

Module B — Drying Room Real-Time Monitoring

  • Real-time display of carts currently in each drying room and their status
  • Automatic calculation of estimated completion time based on configured target drying duration
  • Automatic expiry alerts to prevent over-drying or failure to retrieve on time

Module C — Sampling & Inspection Workflow

  • Configurable sampling rules to reduce unnecessary inspections
  • Lab technicians enter water activity readings directly; system auto-judges Pass/Fail
  • Inspection results reported in real time — no need to wait for end-of-shift roll-up

Module D — Non-Conforming Product Disposition Workflow

  • Failed batches immediately trigger a disposition workflow: Re-test / Re-dry / Ambient dry / Scrap
  • Disposition decisions are fully recorded with a complete audit trail

Module E — Management Dashboard

  • Real-time overview: batches currently drying, pending inspection, today's pass rate, batches requiring attention
  • Qualified batches released to packaging queue with a single click

Module F — Complete Batch Traceability

  • Any batch is fully traceable: Production batch → Sub-lot (cart) → Sampling record → Inspection result → Disposition → Release time
  • Provides a complete evidence chain when inter-departmental disputes arise

Module G — Product & Inspection Standard Management

  • Create products and customize basic information
  • Define inspection items and pass/fail ranges per product (dual-limit or upper-limit-only modes)
  • Configurable manual-review zone: borderline results escalated to a supervisor

Module H — Packaging Release Queue New

  • Wendy (Packaging Dept Leader) logs in to view a complete list of all QC-released carts
  • Supports multi-select and batch Dispatch — confirm multiple carts in one action
  • Release records are linked to corresponding work orders and inspection results; packaging and warehouse data stay in sync in real time

  • QC and packaging teams share a single real-time data platform, eliminating the information black hole
  • Full batch traceability ensures inter-departmental disputes are backed by verifiable records

Add-On Option: IP Access Control

This section describes an optional add-on service, not included in the Phase 1 base price. It may be purchased separately as needed.

Overview

Using Vercel Edge Middleware, access to the system is restricted to a whitelist of approved IP addresses. Requests from any IP not on the whitelist are blocked at the edge before the page loads — this cannot be bypassed by end users.

Use Cases

  • NPIC's factory network has a static IP — add it to the whitelist directly
  • If no static IP is available, a VPN can be configured for relevant staff; all access routes through the VPN's fixed exit IP, which is then whitelisted
  • Multiple IPs can be whitelisted (e.g., factory, office, management home) — NPIC IT manages additions and removals via the Vercel dashboard

Deliverables

  • Vercel Edge Middleware IP whitelist implementation (including test verification)
  • IP management documentation for NPIC IT self-service maintenance
