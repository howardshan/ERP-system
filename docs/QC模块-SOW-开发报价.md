## 1. 文档目的

本 SOW 用于向客户说明 QC 模块的 **交付范围、功能模块、估算工时及费用**，作为商务报价与合同附件。范围以两次客户会议对齐需求为准；带 ⚠️ 标记的条目在客户书面确认前为 **预估值**，确认后超出本合同范围的，通过变更单处理。

---

## 2. 项目概述

### 2.1 业务目标

在现有 ERP 之外，建设 **制造现场 QC + 批次追溯** 能力，覆盖：

- 烘干后检验结果 **实时可见**（替代 Excel 汇总延时）
- 烘干子批 **全流程状态可追踪**（进房 → 待检 → 检验 → Hold/放行 → 包装）
- **SKU 烘干时长控制**与逾期提醒
- **任务型 Dashboard**（六类 WIP + 等待打包数量）
- **系统随机抽样**与 **级联 Reject / Pass 传播**
- **企业微信 / 邮件** Hold 与关键质量事件告警
- **SKU 维度**平均烘干时长、通过率等统计
- **QC 工作量 3 天预测**（启发式，非 MRP 排程）

### 2.2 交付形态


| 项    | 说明                                                  |
| ---- | --------------------------------------------------- |
| 应用形态 | Web 应用（平板优先），集成于现有 ERP 模块体系                         |
| 技术栈  | React + Vite + Supabase（PostgreSQL RPC），与主 ERP 工程一致 |
| 部署   | 客户网络环境内 Supabase / Docker（具体方式启动阶段确认）               |
| 界面语言 | 默认英文                                                |


### 2.3 现有原型说明

团队已完成 QC **核心流程可运行原型**（烘干子批、烘干房网格、检验录入、Hold 处置、基础看板、追溯等）。本合同金额为在原型基础上完成 **规则对齐、两次会议需求补全、测试与上线** 的 **固定开发包**，而非从零建设全套 MES。

## 3. 工时与报价说明

### 3.1 估算口径

各模块人时 **包含**：

- 与需求文档及客户确认项的对齐
- 数据库迁移、业务逻辑、前端页面、权限配置
- 模块内自测与缺陷修复
- 集成测试与 UAT 支持（按模块分摊）

各模块人时 **不包含**（见 §10 排除项）：

- 客户侧硬件采购、车间网络改造
- 现 ERP 厂商接口开发（若 ERP 方不配合）
- 长期运维与 7×24 值守

### 3.2 计价方式

```
模块费用 (USD) = 模块估算人时 × 100 USD/人时
```

所有金额均为 **美元（USD）**，未含税；税费按合同约定另行处理。

## 8. 验收标准

验收以《QC 模块起步与设计指南》§7 KPI 及下列要点为准：


| #   | 验收项       | 标准                                              |
| --- | --------- | ----------------------------------------------- |
| 1   | 检验闭环      | 烘干出房 → 系统指派取样 → 录入 → 判定 → Hold/Pass 全流程可在系统完成   |
| 2   | Dashboard | 六类 WIP 队列及 **等待打包** 计数准确；管理层 **≤15 分钟** 可见新检验结果 |
| 3   | 随机抽样      | 按 SKU 规则 **100%** 由系统指派取车子批                     |
| 4   | 级联 Reject | 抽中样 Fail 后，按签字规则 **同组关联子批级联 Reject/Hold**       |
| 5   | 烘干时长      | SKU 标准时长、子批倒计时与逾期可见                             |
| 6   | 判定模式      | 至少支持 **双限 + 仅上限** 两类（按客户提供清单配置）                 |
| 7   | 消息通知      | Hold / 级联 Reject 触发 **企业微信或邮件** 通知（配置完成后）       |
| 8   | SKU 统计    | 可按 SKU 查询平均烘干时长、通过率（默认时间窗启动阶段约定）                |
| 9   | QC 预测     | 提供未来 **3 天** QC 待检工作量预测视图                       |
| 10  | 追溯        | **≤10 分钟** 可导出生产批 → 子批 → 检验 → 处置链路              |
| 11  | Excel 汇总  | 例行 Excel 汇总 **0 次/班**（由 Dashboard 替代）           |


详细 KPI 数值（如 Hold 处置时限、级联范围）在启动阶段与客户写入验收附件。

## 9. 假设与依赖

1. 客户提供 **启动阶段必需输入**：SKU 清单、检验项与上下限/仅上限对照表、各 SKU 抽样间隔 N、标准烘干时长、**级联 Reject / Pass 传播规则**书面确认。
2. 客户提供 **企业微信 Webhook 和/或 SMTP 邮件** 配置信息（用于 M10 告警）。
3. 客户指定 **业务对接人（质量）+ IT 对接人各 1 名**，反馈周期 **≤3 个工作日**。
4. 车间具备 **稳定 Wi-Fi 或 4G**，QC 终端为平板或 PC 浏览器（Chrome/Edge 最新版）。
5. 部署环境在启动阶段确定；客户负责服务器/域名/SSL（若自建）。
6. 现 ERP **不被替换**；本合同 ERP 集成以 **CSV 导入导出** 为主（不含双向 API）。
7. 交付物为 **可运行软件 + 操作文档 + 规则确认纪要**。
