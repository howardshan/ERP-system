# 宠物零食制造 QC 模块 — 起步与设计指南

> 本文档整理自项目启动阶段的战略讨论，聚焦 **从质量控制（QC）环节切入** 的专用制造数字化系统。

## 1. 背景与问题定义

### 1.1 客户现状

- 已在使用一套 ERP，在 **财务、采购、销售** 等方面基本可用。
- **生产制造、车间、QC、批次追溯** 能力薄弱，满意度低。
- QC 环节以 **纸质记录为主**，事后由管理人员 **手工录入 ERP**，导致：
  - 数据滞后（T+0 现场 → T+1 或更晚才进系统）
  - 库存/放行状态与现场真实质量不一致
  - 追溯、客诉、召回响应慢
  - 管理看板无法反映「当前产线真实状态」

### 1.2 行业特征（宠物零食 / 食品级制造）

| 特征 | 对系统的要求 |
|------|----------------|
| 强批次管理 | 原料批、生产批、成品批全链路关联 |
| 强 QC | 来料 / 过程 / 成品多节点检验 |
| 强追溯 | 由成品批反查原料批、产线、人员、时段 |
| 强合规 | 检验记录留存、审计追踪、不合格品处置留痕 |
| 现场环境 | 手套、油污、少打字、多扫码/点选 |

### 1.3 核心问题（一句话）

> **生产现场质量数据没有实时数字化**，而不是「缺少一套新财务系统」。

## 2. 产品定位与边界

### 2.1 我们做什么

**「QC + 生产现场数据采集平台」**（可演进为 MES-lite），首要目标：

- 现场 **实时录入** 检验结果与关键工艺参数
- **批次（Lot/Batch）** 全链路可追溯
- **Hold / Release** 等处置流程在线闭环
- 管理端 **实时看板**（合格率、Hold 数量、缺陷趋势等）

### 2.2 我们暂不做什么（Phase 1）

- 不替换现有 ERP 的总账、应收应付、税务
- 不做完整 MRP、高级排程（留给后续阶段）
- 不追求「大而全 ERP」第一版

### 2.3 与现有 ERP 的关系（Overlay 策略）

```
┌─────────────────────────────────────┐
│  新系统：制造 / QC / 追溯（真相源）   │
│  · 现场检验、Hold/Release、批次事件   │
└──────────────┬──────────────────────┘
               │ 同步（API / 文件 / 人工确认）
               ▼
┌─────────────────────────────────────┐
│  现有 ERP：订单、库存账务、采购、财务  │
└─────────────────────────────────────┘
```

**开工前必须明确：**

- 物料、客户、供应商等 **主数据以谁为准**
- 检验合格后 **库存/可发货状态由谁更新**
- Hold 是否必须 **回写或锁定** 现 ERP 中的库存

## 3. 推荐实施路线图

| 阶段 | 周期（参考） | 目标 |
|------|-------------|------|
| **Phase 0：发现与映射** | 2–4 周 | 现场调研、流程图、数据字典、集成边界、MVP 范围签字 |
| **Phase 1：QC 数字化 MVP** | 8–12 周 | 替代纸质 QC 录入、实时数据、批次追溯、Hold/Release、基础看板 |
| **Phase 2：生产执行衔接** | 后续 | 工单、报工、产量/损耗、与 QC 联动（未检不可入库等） |
| **Phase 3：库存 / MRP 薄层** | 后续 | 与现 ERP 协同的库存建议、领料、完工 |
| **Phase 4：财务与 ERP 深度集成** | 长期 | 批次成本、报废金额、凭证/接口对接现 ERP 或财务软件 |

## 4. 如何起步（Phase 0 行动清单）

在写业务代码之前，按顺序完成以下事项。每一项未完成，都会在设计或开发阶段返工。

### 4.1 现场调研（必做）

| 活动 | 时长建议 | 产出 |
|------|---------|------|
| 跟生产线至少 1 个完整班次 | 1 天 | 过程 QC 触点时间、记录方式、谁签字 |
| 跟 QC 员（来料 + 成品） | 1 天 | 检验项清单、抽样规则、纸质表单拍照 |
| 跟仓库（收料、发货） | 0.5–1 天 | 批次号如何生成、贴标、与 ERP 录入关系 |
| 访谈生产经理 / 质量负责人 | 2–4 小时 | 痛点排序、审批链、合规要求 |
| 访谈 IT 或 ERP 管理员 | 2 小时 | 现 ERP 品牌版本、接口、主数据导出方式 |

**原则：** 不仅听管理层，必须听 **QC 技术员、线长、仓库文员**——他们才是日常用户。

### 4.2 交付物（Phase 0 结束时应具备）

- [ ] **端到端流程图**（从原料入库 → 来料检 → 投产 → 过程检 → 包装 → 成品检 → 入库 → 发货）
- [ ] **QC 节点清单**（每个节点：触发条件、检验项、频次、责任人、现行表单）
- [ ] **批次号规则说明**（编码规则、在哪些环节打印/扫描）
- [ ] **角色与权限矩阵**（谁可录入、谁可 Hold、谁可 Release）
- [ ] **与现 ERP 集成说明**（同步字段、频率、冲突处理）
- [ ] **MVP 范围文档 + 验收 KPI**（客户签字或邮件确认）
- [ ] **主数据初版清单**（物料、供应商、产线、检验模板）

### 4.3 内部准备

- [ ] 确定技术栈与部署方式（云 / 本地、车间网络、是否需离线）
- [ ] 确定第一版终端（iPad / Android 平板 / 手机 PWA / 扫码枪）
- [ ] 确定项目对接人（客户方质量 + IT 各一人）

## 5. QC 模块设计所需必要信息

> **本节是文档核心。** 设计 QC 模块前，应尽可能填满下列信息；空白项代表设计风险，需在调研或原型阶段补齐。

### 5.1 业务目标与成功标准

| 编号 | 问题 | 填写 / 备注 |
|------|------|------------|
| B-01 | 客户最想解决的 Top 3 痛点是什么？（延迟录入 / 追溯慢 / Hold 不及时 / 其他） | |
| B-02 | Phase 1 结束后，客户如何评判「成功」？（可量化） | |
| B-03 | 主要销售市场（内销 / 出口美国 / 其他）及未来 2 年合规重点？ | |
| B-04 | 是否已有客户或零售商审厂要求（如 Costco、SQF、BRCGS）？ | |
| B-05 | 预计同时在线使用的 QC/现场用户数？ | |

### 5.2 生产与 QC 流程（流程映射）

为 **每一条产品线或车间**，描述完整路径，并标注 QC 节点。

| 编号 | 问题 | 填写 / 备注 |
|------|------|------------|
| F-01 | 有哪些产品线 / 车间 / 产线？各线工艺是否差异大？ | |
| F-02 | **来料检验（IQC）**：哪些物料必须检？哪些可免检？ | |
| F-03 | IQC 触发时机：到货即检 / 入库前 / 其他？ | |
| F-04 | **过程检验（IPQC）**：有哪些工序节点（烘干、混合、金属检测等）？ | |
| F-05 | IPQC 频次：每批首件 / 每 30 分钟 / 每班 / 其他？ | |
| F-06 | **成品检验（FQC）**：在包装前、包装后还是入库前？ | |
| F-07 | 是否存在 **驻厂检 / 第三方检 / 客户批检**？ | |
| F-08 | 检验不合格时，现场 **标准处置流程**（隔离、返工、报废、让步接收）？ | |
| F-09 | 是否存在「未过某道 QC 不允许继续下道工序」的硬规则？具体是哪些？ | |
| F-10 | 留样规则：是否留样、留多久、如何编号与追溯？ | |

**建议产出：** 一张带 QC 菱形节点的 **泳道流程图**（角色：QC、线长、仓库、生产、质量经理）。

### 5.3 检验项与标准（检验体系）

| 编号 | 问题 | 填写 / 备注 |
|------|------|------------|
| T-01 | 检验项按什么维度配置？（物料类别 / 具体 SKU / 产线） | |
| T-02 | 每类检验有哪些 **检验项目**？（名称、单位、类型） | |
| T-03 | 每个项目的 **数据类型**：数值 / 是否合格 / 单选 / 多选 / 文本 / 照片 | |
| T-04 | 每个项目是否有 **标准值、上下限、目标值**？谁有权修改标准？ | |
| T-05 | **抽样方案**：固定数量 / AQL / 按比例 / 全检？ | |
| T-06 | 是否需要 **检测设备对接**（温湿度仪、金属检测机、水分仪等）？接口形式？ | |
| T-07 | 是否需要 **COA（分析证书）** 与来料批关联上传？ | |
| T-08 | 现行纸质表单（请拍照归档）：字段名、必填项、签字栏 | |
| T-09 | 检验项是否会随 **配方版本** 变化？如何版本管理？ | |

**设计提示：** 系统应支持 **检验模板（Template）** + **检验单实例（Record）**，避免每个 SKU 硬编码表单。

### 5.4 批次、追溯与标识

| 编号 | 问题 | 填写 / 备注 |
|------|------|------------|
| L-01 | **原料批号** 由谁生成？规则是什么？ | |
| L-02 | **生产批号** 由谁生成？是否与工单号一致？ | |
| L-03 | **成品批号** 规则？是否与生产日期、产线、SKU 相关？ | |
| L-04 | 现场是否已有 **条码 / 二维码**？格式（Code128、QR）？ | |
| L-05 | 标签在哪些环节打印？用什么软件/打印机？ | |
| L-06 | 追溯要求：成品批 → 需反查哪些信息？（原料批、供应商、产线、操作员、设备、时间段） | |
| L-07 | 是否存在 **拼批、拆批、混合批**？如何记录？ | |
| L-08 | 保质期 / 生产日期 / 最佳食用日期 如何计算与校验？ | |

## 6. Phase 1 MVP 功能范围建议

第一版只做「能替代纸、能闭环、能追溯」，克制范围。

### 6.1 建议包含

| 模块 | 功能要点 |
|------|---------|
| 主数据 | 物料（同步或导入）、供应商、产线、检验模板 |
| 批次管理 | 创建/扫描批次、关联原料批与生产批（基础级） |
| 检验执行 | 按模板创建检验单、数值/合格项/照片、提交与复核 |
| 处置 | Hold / Release、不合格原因、审批（简化版） |
| 追溯 | 按批号正查/反查链条（含检验记录与照片） |
| 看板 | 当前 Hold、今日合格率、逾期未检提醒（基础） |
| 审计 | 操作日志、关键操作电子签名（若客户有要求） |
| 集成 | 至少一种：CSV 导出 或 单接口同步物料/库存状态 |

### 6.2 建议不包含（避免 scope creep）

- 完整工单排程、设备 OEE、复杂 MRP
- 总账、应收应付、税务
- 与所有检测仪器全自动对接（可列 Phase 2）
- 多工厂、多语言（除非合同明确要求）

## 7. 验收指标（KPI）建议

写入合同或里程碑，避免「感觉好用」无法验收。

| 指标 | 现状（调研填写） | 目标（示例） |
|------|-----------------|-------------|
| 检验记录进入系统时效 | 如：次日录入 | 检验完成后 **≤15 分钟** |
| 批次追溯耗时 | 如：2 小时 | **≤10 分钟** 导出完整链 |
| Hold 生效时效 | 如：依赖文员录入 | 判定不合格后 **系统自动 Hold** |
| 纸质表单减少比例 | — | 目标 QC 类型 **≥80%** 电子化 |
| 数据准确性 | — | 抽样核对 **≥99%** 与现场一致 |

## 8. 技术架构原则（概要）

### 8.1 核心设计原则

1. **事件/事务导向**：不仅存最终状态，保留检验提交、Hold、Release 等事件，便于审计与追溯。
2. **批次为中心**：检验单、照片、处置记录均挂载批次 ID。
3. **Mobile / Tablet First**：现场能用是第一优先级；考虑 PWA + 弱网离线。
4. **模板驱动检验**：检验项配置化，减少为每个 SKU 写死页面。
5. **审计默认开启**：谁、何时、做了什么、改前改后（食品行业刚需）。

### 8.2 技术栈（可按团队调整）

| 层级 | 常见选型 |
|------|---------|
| 前端 | React + Vite，平板适配，PWA |
| 后端 | FastAPI / Node.js 等 |
| 数据库 | PostgreSQL / MySQL |
| 部署 | 按客户网络条件选择 Docker 本地或云 |

> 语言栈不如 **数据模型与集成边界** 重要；优先保证批次模型和 ERP 接口契约正确。

### 8.3 核心实体关系（概念）

```
物料 ──< 检验模板
批次 ──< 检验单 ──< 检验项结果
批次 ──< 质量事件（Hold/Release/报废…）
批次 ──> 原料批关联（追溯链）
批次 ──?── 现 ERP 库存/工单 ID
```

## 9. 附录：现场调研问卷模板

调研时可按角色使用以下提纲（录音/拍照需征得同意）。

### 9.1 QC 技术员

1. 你每天做哪几类检验？各花多少时间？
2. 现在用哪些纸质表？每张表必填项是什么？
3. 检验标准从哪里查？有没有经常变更？
4. 不合格时你下一步做什么？找谁签字？
5. 如果给你一台平板，你最希望少填哪些字？
6. 车间哪里信号最差？

### 9.2 线长 / 生产主管

1. 开工前必须完成哪些 QC？没完成能否开工？
2. 过程检多久做一次？谁提醒？
3. 产线换型、换 SKU 时批次号怎么处理？
4. 你最常因为 QC 数据不准确遇到什么问题？

### 9.3 仓库

1. 到货时批号怎么贴？和 ERP 入库单如何对应？
2. Hold 的货放在哪里？系统里现在能否看出 Hold？
3. 发货前是否核对成品检状态？

### 9.4 质量经理

1. 当前最大质量风险是什么？
2. 审厂或客诉时，追溯要准备哪些材料？准备多久？
3. 哪些人必须有 Release 权限？
4. Phase 1 三个月内，你最想看到哪三个数字？

### 9.5 ERP / IT

1. ERP 品牌版本、模块清单、是否有 API？
2. 能否提供物料、库存、工单样例导出？
3. 未来是否接受「QC 系统驱动库存状态」？
