1. Overview

This proposal outlines the scope of Phase 2 of the NPIC digital operations platform. Building on the QC inspection system established in Phase 1 (Modules A–I), Phase 2 focuses on digitizing the production floor — replacing the daily Excel log and paper-based cart records with a real-time, role-controlled system, while integrating directly with Sage 100 to eliminate manual double-entry.

Phase 2 introduces five modules (A–E):

3. System Architecture

Phase 2 is built on the same technical stack as Phase 1 and integrates directly into the existing NPIC platform. New modules share the same user authentication, barcode scanning, and export infrastructure.

Role & Access Design

RoleModules AccessibleMFA Required?
EmployeeModule B — Station Mode (sign-in/out, cart entry)No
SupervisorModules A, B, C, D — full entry, review, and reportingNo (standard operations)
ManagerAll modules + restricted actions (work order close)Yes — Microsoft Authenticator
AdminFull system access including Sage 100 integration configurationYes — Microsoft Authenticator

Key Design Principles

  • Employee Mode — Lightweight production-floor interface. Badge scan or employee ID sign-in. No MFA required for daily entries.
  • Supervisor Mode — Full batch data entry matching the layout of the existing Excel form.
  • Privileged Actions — Work order closing requires Microsoft Authenticator verification (inherited from Phase 1, Module I).
  • SKU-Configurable Fields — Required fields (weight vs. quantity, cart tray layers, work order fields) are pre-configured per product SKU.
  • Half-Cart Linking — Physical cart continuations are bound in the production stage (Module B); all downstream modules inherit the association automatically.

4. Module Specifications

Module A  Production — Supervisor Data Entry

Replaces the existing Excel daily report. Supervisors enter batch records for all workers and carts in a given shift or period. This mode is used by teams that do not have individual barcode scanning stations.

Data Captured
  • Employee names and IDs
  • Machine assignments
  • Work order numbers (工单号)
  • Primary material codes — 一级单 (raw material identifiers)
  • Cart numbers, tray layers, and quantity or weight per cart
  • Shift hours and total labor time
Outputs
  • Auto-calculated per-employee productivity metrics (fed into Module D reporting)
  • Records linked to work orders for Sage 100 sync via Module E
Module B  Production — Employee Station Mode

A streamlined, employee-facing interface for real-time production logging at the machine or line level. Designed for shared tablets or terminals on the production floor. No MFA required for standard entries.

Sign-In / Sign-Out
  • Employees sign in before the shift by entering their employee ID or scanning their badge. Multiple employees can sign in simultaneously on the same machine.
  • A digital signature is required at sign-out.
  • The shift closes when all employees assigned to that line have signed out.
Cart Entry
  • On the first cart, the employee inputs the primary work order number(s). Subsequent carts auto-populate from the last entry — a confirmation button must be pressed before proceeding; manual input can override the default at any time.
  • One cart can be associated with multiple primary work orders.
  • Required fields (weight or quantity, tray layers) are pre-configured per product SKU.
  • Cart number and tray layer count are recorded for each entry.
Half-Cart Continuation
  • For the first cart of a session, the employee must select one of the following — mandatory, cannot be skipped:
  • Standalone cart — a new, independent production run.
  • Half-cart (continuing) — a physical continuation of a previous session's cart. The system links this cart to the prior cart number. All downstream modules (dry room, reporting, Sage 100 sync) inherit this association automatically — no re-selection required anywhere else in the workflow.
Work Order Close — Restricted (MFA Required)

Because actual production output is typically less than the theoretical BOM quantity, a work order close function is provided. This action requires special permission and Microsoft Authenticator verification.

OptionDescription
No RemainderAll materials fully consumed. Work order closes cleanly.
Scrap RemainderRemaining raw material written off. System auto-distributes unused quantity evenly across finished carts for accurate material accounting.
Retain RemainderRemaining material reserved for a future run. No adjustment to cart records.
Module C  Dry Room Operations

Tracks carts through the dry room with production-record linkage. No half-cart re-selection is required — the association established in Module B carries through automatically.

  • Scan-In: Operator scans the cart barcode on entry. The system retrieves the cart's production record (including any half-cart binding) automatically from the barcode and records the entry timestamp.
  • Scan-Out: Records exit timestamp when the cart leaves the dry room.

Additional dry room workflow details (redry queue, time/temperature logging) will be confirmed with the NPIC team during the requirements phase.

Module D  Data Summary & Reporting

Aggregates all production data recorded in Modules A and B into structured reports and exports, replacing the manual compilation currently done from the Excel file.

Shift & Daily Reports
  • Auto-generated shift summary: total carts, total quantity/weight, work hours, and headcount per shift
  • Daily production summary across all machines and work orders for a given date
  • Per-work-order completion report: planned vs. actual quantities, material reconciliation status
Productivity Reports
  • Per-employee output (carts completed, weight/quantity produced, hours worked) over any date range
  • Machine utilization: active hours vs. idle time per line
  • Trend view: week-over-week or month-over-month output by product or work order
Data Export
  • All reports exportable by date range or product SKU
  • Export formats: Excel and PDF
  • Leverages the Phase 1 export framework for consistency
Module E  Sage 100 Integration

Connects the NPIC system directly to the Sage 100 database for real-time data exchange — eliminating the manual re-entry of work orders, inventory adjustments, and production completions between the two systems.

Sync Design
  • Real-time pull: Work order and item master data is fetched from Sage 100 on demand when a user opens a new record, ensuring the latest data is always used.
  • Triggered push: Production completions and inventory adjustments are posted to Sage 100 at defined workflow milestones (work order close) — not on a polling schedule — minimizing conflicts.
  • Error handling: If a push to Sage 100 fails, the system queues the record and retries automatically. Supervisors are notified of any items requiring manual resolution.

Detailed Notes

7.1 Eliminate manual Excel daily report. Today, each supervisor manually consolidates the shift's production in Excel and compiles a daily report. Once the system is live, the report is generated automatically from real-time shop-floor capture, with no manual entry. At roughly 2 hours/day per supervisor, across 10 supervisors × 22 working days, this saves about 440 labor-hours/month (≈2.5 FTE); the daily report shifts from “available next day” to “available in real time.”

7.2 Real-time cart-level data. Each cart generates a timestamped record the moment it is completed — no after-the-fact reconstruction from memory. Backfilling approaches 0; per-cart traceability reaches 100%, with timestamps accurate to the minute.

7.3 Work-order-level raw-material usage tracking. Actual issuance and consumption are recorded per work order. Current pain point: material from one work order is often used on another, so actual usage cannot be attributed accurately. Binding material to the work order eliminates the distortion caused by cross-work-order material mixing, allowing each order's actual usage / yield to be computed accurately — supporting cost accounting and over-consumption detection.

7.4 Accurate cart count & timing tracking. Each cart's actual output quantity plus its start/finish timestamps are captured in real time. Current pain point: employees may deliberately omit or under-report carts, so the actual number produced does not match the registered number. With real-time counting the two match. Actual cart count = registered cart count, under-reporting drops to zero, output-registration accuracy rises to 100%, and an average takt time (minutes/cart) can be derived for capacity estimation and scheduling.

7.5 Post-production waiting-time logging. The waiting time after each cart finishes production (e.g., waiting to enter the next step / the drying room) is recorded, building up data that provides a basis for future process optimization and efficiency improvement. The average waiting time (minutes/cart) can be measured and used as a quantified baseline and tracking metric for improvement.

7.6 No manual entry of workstation & labor hours. Workstation assignment and working hours are captured by the system, replacing paper/Excel manual entry. The labor-hour saving here is already included in the 7.1 daily-report saving and is not double-counted; the value lies in improved completeness and accuracy of the labor-hour / workstation data (eliminating missing and incorrect entries).

7.7 Employee cart-level productivity. Productivity is computed automatically and accurately per employee × cart (carts/shift, units/labor-hour) — finer-grained, more objective, and real-time than the current approach. Data granularity reaches “person × cart”; productivity dispersion across teams/individuals can be compared, supporting performance evaluation and staffing decisions. (An objective baseline already exists; this item makes the data more accurate and more granular, not something created from nothing.)

7.8 Work-order-level real-time production tracking. The production progress/status of each work order is tracked in real time, so planning and production directors can monitor and plan more accurately based on actual performance (scheduling, capacity, delivery dates). Work-order progress is visible in real time (100% coverage), shifting from “after-the-fact roll-up” to “real-time command.”

7.9 No manual filling of per-cart records & reports (line workers). Today, line workers must manually fill in each cart's production record and statistical reports; once the system is live, data is captured in real time on the shop floor and workers no longer fill anything in manually. At 1 minute/cart × 6 carts/shift × 3 shifts/day × 40 production lines = 720 minutes/day (12 labor-hours/day), across 22 working days, this saves about 264 labor-hours/month (≈1.5 FTE).
